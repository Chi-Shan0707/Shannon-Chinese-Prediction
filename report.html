A tiny Shannon experiment on Chinese

让小模型猜中文：从《三国演义》到疯狂星期四

同一套下一 token 猜测：维基百科一路平滑，方言小说开始打滑，疯狂星期四在固定短语和梗脚本之间露出缝。

Qwen3-0.6B Base · 11 类中文文本 · 59,496 个预测步

项目仓库：github.com/Chi-Shan0707/Shannon-Chinese-Prediction/

我先拿“疯狂星期四”试水。前缀是“因为疯狂”，模型下一步直接押“星期四”，Rank 1；前缀换成“肯德基疯狂”，它还是马上摸到“星期四”。可一旦句子走到“V我50吃一顿……”，“肯德基”反而不见了。

口号会背。笑点要另算。

Shannon 当年的猜字实验有一种手工感：猜下一个字母，错了再猜，文本的秩序就在这些错误里慢慢浮出来。中文会把这个游戏弄得更毛糙：下一步到底算一个字、一个词、一个成语、一个人名、一个梗，还是 tokenizer 切出来的一片边角料？

我就把猜的人换成 Qwen3-0.6B，让这个小模型沿着中文文本一格一格往前猜。

别急着给“中文熵”一个总分

总分太粗。真正好看的，是模型在哪一瞬间突然笃定，又在哪一瞬间开始散票。

Qwen3-0.6B 每走到一个位置，都会交出一张概率表：它押哪些 token，押得有多重。顺滑的续写会把概率压到前排；方言、文学腔、地方口吻、反转包袱，会把候选摊开。

好玩的地方在这儿：概率表像一盏斜光，照出纸面的纹理、折痕和毛边。

语体会留下痕迹。 维基百科和新闻一路平滑；方言小说明显更颠。原本靠感觉说的“语体差异”，在这里变成了曲线。
套路不等于确定。 《三国演义》最容易进 top-1000，却没有拿到最低熵。正确答案常在附近，但旁边还站着一圈像样的候选。
梗会裂开两层。 “疯狂 → 星期四”几乎自动完成；“吃 → 肯德基”会卡住。前者是固定搭配，后者要接上中文互联网的暗号。
上下文会压弯曲线。 更多上下文当然有用，最值钱的是前几十个字。再往后，收益开始变薄。

读文本时那些很难说清的感觉——顺、涩、熟、拐弯——被拆成了概率上的动作。

游戏规则：猜中就往前走

每段文本都被切成很小的预测步。每一步只看两件事：真实下文有没有出现在模型的候选里；模型押这些候选时，到底是集中还是分散。

脚本检查 top-1000 个下一 token。只要某个候选正好贴上真实剩余文本的开头，就算命中，并按这个 token 的长度前进；如果 top-1000 里都贴不上，就记一次 miss，只挪一个字符。

核心记录的是模型下一 token 分布的熵：

\[H(q)=-\sum_t q(t)\log q(t).\]

代码用自然对数，所以单位是 nats per prediction step。它适合做这篇里的横向比较：同一个模型、同一套游戏规则，看不同文本把概率表搅成什么形状。

我故意把语料混得很杂

维基百科、新闻、古典小说、武侠、现代文学、方言小说、反转段子、疯狂星期四文案，全都扔给同一个小模型。语料越杂，模型越容易露出脾气。

维基百科是平路，方言小说是石子路

这个结果不意外。维基百科和新闻本来就是语言模型的柏油路：定义、专名、时间地点关系、话题标签、报道公式，全都很规整。方言小说像石子路，地方声音、口语节奏、拐来拐去的说法都会让下一步变得更滑。

“方言味儿”和“口语感”平时容易被说成玄学；在这里，它们直接表现为更散的下一 token 分布。

《三国演义》最会骗人

《三国演义》的 top-1000 命中率最高：95.0%。但它没有拿到最低平均熵。这个组合很耐看。

原因大概在“附近”和“唯一”之间。套路能把正确续写送到附近：“却说”“话说”“玄德曰”、战阵描写、官职称谓、章回体机器，都会帮模型摸到街区；可街区里还有很多门，概率未必会像维基百科那样狠狠压到一个 token 上。

模型找得到，不代表它心里只有一个答案。

疯狂星期四：会背口号，不一定懂转弯

中文互联网里这个动作太熟了：前面可以正常、煽情、狗血、悬疑，最后突然拐到“今天疯狂星期四，V我50”。我直接去戳这个暗门。

模型把“疯狂星期四”吃成了一整块。但“吃 → 肯德基”要难得多。“吃”会打开整个食物宇宙；最后落到肯德基，要靠那套转账、星期四、KFC 的暗线，而不只是补全搭配。

这个小测试把可预测性劈成两半：一边是背熟的短语，一边是闻出铺垫。模型会接上口号，但不总能闻到笑点。

上下文：前几十个字最值钱

Shannon 实验真正迷人的地方是上下文：前文多给一点，猜测会容易多少？我挑了四部长文本，把上下文窗口从 \(N=5\) 拉到约 \(N=200\)，再用一个简单衰减式拟合：

\[H(N)=H_\infty+aN^{-b}.\]

我更在意曲线怎么弯，而不是参数小数点后几位。前几十个字最值钱，能迅速买到确定性；再继续加一百个字，当然还有用，但没有前面那么划算。曲线慢慢变平的样子，正好把“上下文的边际收益”画了出来。

口径：这里看的是 token 熵

这几个数字的口径要摆清楚：它们是模型侧的 token 熵，衡量 Qwen3-0.6B 在每一步对下一 token 的分布有多分散。

  • 单位：nats per prediction step，不是 bits per character。
  • 预测器：一个小模型，不是“中文真实分布”。
  • 粒度：token-match，不是字符级猜测。

要走字符级路线，就得把 token 概率折回下一个字符的概率：

\[P(c\mid C)=\sum_{t:\operatorname{prefix}(t)=c} P(t\mid C).\]

那条路更适合和旧的 bits-per-character 工作对齐。眼下这个 token 版本先做一件更朴素的事：把文本里那些不平整的地方照出来。

语言学藏在这些卡顿里

分数之外，更值得盯的是这些语言问题：

  • 哪些中文让模型犹豫？
  • 套路化写法是让模型更自信，还是只是把正确答案放到附近？
  • 模型到底是认识短语，还是认出了梗？
  • 上下文给到多长，收益才开始变薄？

一个总熵数字会把这些差别压扁。可预测性有很多形状：一段文本可以很套路，让正确续写经常在附近，但分布仍然不够尖；另一段文本主题普通，却因为声音和句法更毛糙而让模型散开；一个梗可以在局部极其明显，在整体脚本上又突然打滑。

不同中文文本有不同的“可猜”方式。小模型不一定聪明，但它很诚实：哪里顺，哪里卡，它都会在概率表上露出来。
