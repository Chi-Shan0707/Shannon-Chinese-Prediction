汉语文本的可预测性：
一项 Shannon 式 token-match 实验

用 Qwen3-0.6B Base 作为自动预测器，观察语体、网络梗与上下文长度如何改变模型不确定性
复旦大学 · 语言科学导论 · 课程项目报告 · 2026年5月

摘要　Shannon (1951) 通过人类逐字母猜测英语文本，展示了上下文如何压缩语言的不确定性。汉语常用字规模远大于英语字母表，若要求人类在每个位置从数千候选字中猜测，实验成本极高。本文以 Qwen3-0.6B Base 作为自动预测器，对 11 类汉语文本进行 token-match 预测，共完成 59,496 个预测步。实验最稳健的发现是相对排序：维基百科最可预测（平均预测熵 \(H=3.53\) nats/step），《俗世奇人》最高（\(H=4.48\) nats/step）；《三国演义》的 Top-1000 匹配率最高（95.0%），但平均熵并非最低，说明“套路性”并不只表现为整体分布更尖锐。定点测试还显示，模型能稳定预测“疯狂→星期四”这类固化搭配，却难以稳定预测“吃→肯德基”这类依赖梗语境的反转。四部长文本的扩展实验显示，\(H(N)\) 随上下文长度增加而近似幂律下降，提示上下文信息存在边际收益递减。

口径说明　代码使用自然对数计算 \(-\sum_{t} q(t)\log q(t)\)，因此本文报告的主指标是模型下一 token 分布熵，单位为 nats/预测步。它适合比较同一模型、同一实验设计下不同文本的可预测性，但不等同于严格的字符级 bits/char 熵率；与孙帆、孙茂松的 \(H_0=9.62\) bits/char 或 Takahashi 与 Tanaka-Ishii 的 bpc 结果只能作方向性比较。

1 研究问题

Shannon 的核心问题不是某个孤立符号有多难猜，而是上下文如何逐步降低下一个符号的不确定性。给定前 \(N\) 个符号，预测下一个符号的条件不确定性记为 \(H(N)\)；当 \(N\) 增大时，语言中的重复、搭配、句法和语义依赖会共同降低 \(H(N)\)。

汉语为这一问题提供了一个有趣场景。字符候选空间很大，人类受试者很难像英语逐字母实验那样逐位置猜测。语言模型则可以在每个位置给出完整词表上的概率分布。本文并不声称模型就是人类读者，而是把它作为一个可重复、可记录概率分布的自动预测器，用来观察不同汉语语体的相对可预测性。

3 方法

3.1 Token-match 游戏

给定一段文本，从第 5 个字符之后开始预测。每一步把已知前缀输入模型，模型在完整词表上计算下一 token 的概率分布；实验只在概率最高的 \(K=1000\) 个 token 中寻找命中。若候选 token \(t_k\) 是原文剩余部分的前缀，则记录排名 \(k\)、概率和分布熵，并按该 token 覆盖的字符数前进；若 top-1000 内无命中，则记录 miss 并前进 1 个字符。

这个设计把 Shannon 的逐字母猜测推广为逐 token 猜测。分词器词表来自大规模语料中的高频子串和共现模式，因此多字符 token 的命中本身反映了一部分汉语搭配结构。不过，token 级结果不能直接等同于字符级熵率；若要严格估计字符级 \(H(N)\)，还需要把所有以同一字符开头的 token 概率聚合为字符概率。

3.2 模型与数据

模型为 Qwen3-0.6B Base，实验过程中不更新参数。数据涵盖百科、新闻、科普、网络梗、章回体、武侠、现代文学、方言文学和历史散文等 11 类文本。主实验上下文范围为 \(N=5 \to 50\)，扩展实验对四部长文本延伸至 \(N=200\)。

结果的主线很清楚：百科和新闻的平均预测熵最低，方言文学最高。正式文本术语密度高、搭配固定，模型更容易把概率集中到少数候选；文学文本尤其是带方言色彩的小说，口语化表达、叙述节奏和局部创新更多，预测分布更分散。

《三国演义》是最有趣的例外。它的平均预测熵为 4.00，并非最低，但 Top-1000 匹配率达到 95.0%，为所有类别最高。这说明章回体的固定套话和叙事模板能让正确 token 经常进入候选集，却不一定让整个概率分布像百科文本那样高度尖锐。

5 网络梗与反转点

6 上下文长度实验

为了观察 Shannon 式条件熵链，对《三国演义》《白鹿原》《人类简史》《天龙八部》四部长文本进行扩展实验，把上下文长度延伸到 \(N=200\)，并按对数间距分箱后拟合：

7 讨论

本实验的价值不在于直接宣布“汉语熵率是多少”，而在于建立一个可重复的预测框架，并得到一组有解释力的相对结果。维基百科、新闻、科普等说明性文本更可预测，方言文学和现代小说更不可预测；网络梗中固定搭配可被准确命中，反转处则更依赖文化语境；上下文长度带来的收益呈递减趋势。

需要区分两个相关但不同的指标。模型分布熵 \(-\sum_{t} q(t)\log q(t)\) 衡量模型自身对下一 token 的不确定性；真实目标 token 的 \(-\log q(t^*)\) 才是 surprisal。本文主要分析前者，因此只能谨慎地讨论“模型预测不确定性”。若要把结果与阅读时间或眼动数据相连，需要记录真实目标的 surprisal，并与人类行为数据做对应。

同样，本文不把 \(H_\infty\) 与字符级 \(H_0\) 相除来给出精确冗余度。严格的汉语冗余度需要在同一粒度和同一单位下比较：例如把 token 概率聚合到下一个字符，使用 bits/char 计算 \(H(N)\)，再与字符级 \(H_0\) 比较。当前结果更适合作为 token 级模型指标，展示方向和结构，而不是作为最终字符熵率。

8 局限与展望

本实验有四个主要局限。第一，只使用单一小模型 Qwen3-0.6B，模型规模和训练语料会影响概率分布。第二，top-1000 token-match 会低估长尾命中，miss 的位置也不能仅凭命中记录还原原文。第三，当前单位为 nats/预测步，不能直接和 bits/char 文献数值合并。第四，语体覆盖仍有限，口语转写、诗歌、法律文书和社交媒体短文本都值得补充。

后续最直接的改进是实现字符级概率聚合：\(P(c\mid C)=\sum_{t:\operatorname{prefix}(t)=c}P(t\mid C)\)。这样可以得到真正的字符级条件熵链，并与 \(H_0=9.62\) bits/char 在同一口径下比较。另一个方向是测试 0.6B、7B、72B 等不同规模模型，观察语体排序是否保持稳定。

9 结论

本文用语言模型复现了一个适合汉语的 Shannon 式预测实验。结论可以概括为三点：第一，不同汉语语体在模型预测不确定性上存在系统差异，百科和新闻更可预测，方言文学更不可预测；第二，固定搭配和网络梗反转的预测难度不同，“疯狂→星期四”远比“吃→肯德基”稳定；第三，随着上下文长度增加，\(H(N)\) 近似幂律衰减，说明上下文信息的边际收益递减。最稳妥的贡献不是给出汉语熵率的最终数字，而是提供一个可复现的实验框架和一组可解释的相对发现。
